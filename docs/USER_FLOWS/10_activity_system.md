# 10. Система активности пользователя

## 1. Название сценария
Система активности пользователя

## 2. Цель сценария
Ввести простую и прозрачную систему накопления очков активности за действия пользователя в приложении CabrioRide. Активность отражает вовлечённость и вклад участника в жизнь клуба.

## 3. Участники и роли
- Пользователь (участник клуба)
- Модератор, админ (просмотр активности, аналитика)

## 4. Принципы работы
- У каждого пользователя есть поле "активность" (`activity`), которое отображается в профиле.
- Очки активности начисляются только за собственные действия пользователя.
- Взаимодействие между пользователями (лайки, респекты и т.д.) отсутствует.

## 5. За что начисляется активность
- Регистрация (создание профиля)
- Заполнение профиля (достижение определённого процента заполнения) весь профиль Х балов. если заполнен частично - соответсвенно дачется часть
- Добавление автомобиля. весь профиль авто Х балов. если заполнен частично - соответсвенно дачется часть
- Добавление события
- Оставление отзыва
- Добавление объекта (гид)
- Участие в событиях
- Другие значимые действия (по решению команды)

> Все значения наград централизованно настраиваются в параметрах `activityRewards` в [config/app.config.ts](../../config/app.config.ts).

## 6. Использование активности
- Активность может использоваться для сортировки участников, отображения в профиле, геймификации.
- В будущем возможно расширение механики (например, доступ к отдельным функциям, привилегии и т.д.).

## 7. UX-детали
- Значение активности отображается в профиле пользователя.
- Можно реализовать визуализацию (например, прогрессбар, бейджи, уровни).
- Нет кнопок для начисления активности другим пользователям.

## 8. Интеграции и API
- Поле `activity` в таблице users (INTEGER, default: 0)
- Все параметры системы активности настраиваются в config/app.config.ts
- По умолчанию при регистрации значение активности = 0, но сразу начисляются баллы за регистрацию и последующие действия.

## 9. Ограничения и лимиты
- Начисление очков только за собственные действия
- минимум активности— 0
- Нет ежедневного сгорания или лимитов на начисление

- Пользователь оставляет отзыв — активность увеличивается
- Пользователь заполняет профиль — активность увеличивается
